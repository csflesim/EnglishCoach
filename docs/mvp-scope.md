# MVP 範圍 (MVP Scope)

> 一個人做得完的範圍。核心是 FSI 四式操練 + 弱點引擎，三頁,其餘都先不做。

---

## 唯一目標

讓這個反射迴圈跑順、自己每天願意用:

```
看到核心句型 → 聽提示連發 → 開口說 → 系統聽你說完 → 給正解與反應速度 → 弱點記錄 → 隔天再練
```

---

## ✅ 要做的 (In Scope)

| 功能 | 說明 |
| :--- | :--- |
| **訓練頁 + 操練 session** | 選句型 → 選模式 →(第二層)→ 操練。替換選句框(~20發)、轉換選句框→選操作(過去/否定/疑問)、擴展每次抽5句最多5層;情境反應關閉 |
| **學習地圖推進** | 完成模式→記錄;單元三式做完=掌握;首頁 Next Up 自動推薦、X/30 進度(localStorage 持久化) |
| **AI 評分(可選)** | 🤖 手動開;說完→STT→評分(對錯/糾錯/弱點);未設金鑰自動退回免費。回合制 ~$5/月 |
| **自由發揮模式** | 回合後 3 題自主產出(無 cue)、也可直接進入;AI 評分+記錄 |
| **後台內容管理** | `/admin` 管理句框 + 單字庫(localStorage);新增即用於操練 |
| **AI 補詞** | 🤖 依分類建議單字(英+中)存入詞庫;結構不變、只豐富變數;每回合隨機抽一批 |
| **單字追蹤 + 複習專區** | 詞庫記使用次數/熟悉度;回合結束勾選不熟→Weak;單字頁卡片複習 |
| **語音輸入 + 反應速度** | 麥克風偵測「開口／說完」,量反應秒數(VAD;備援手動兩鍵) |
| **Echo Loop（可選）** | 正解後加跑 Target→母語→Target 錨點循環(中英雙語播報) |
| **AI 評分** | STT + 評估(準確度/文法/流暢度/反應 + 簡短糾錯) |
| **弱點引擎** | 追蹤弱字/弱句型/慢反應/常錯結構,反覆出現直到掌握 |
| **弱點頁** | 顯示弱點,可點選直接操練 |
| **進度頁** | 練習天數、正確率、平均反應速度、弱點數、估計 CLB |
| **句型庫 + 學習路徑** | FSI 30 單元語言地圖(三週期,受控內容,非 AI 生成) |
| **單字匯入** | 使用者自帶單字清單(帶分類+翻譯),替換變數**從單字庫依分類抓取**,匯入越多內容自動擴增 |

## 🚦 建議實作順序

1. **骨架**:Next.js + Supabase Auth + 資料表 schema + 種子句型(FSI 30 單元起頭幾課)。
2. **核心操練**:句型 → 四式步驟 → 顯示正解(已有 mock 介面)。
3. **接語音**:麥克風 VAD(開口/說完) + 反應速度(已有瀏覽器 VAD);接 OpenAI STT。
4. **接評分**:STT 文字 + 句型 → 評估 prompt → 真實分數與糾錯。
5. **弱點引擎**:答錯/慢反應寫入 `weakness_items`,操練優先載入弱點。
6. **進度頁**:彙總呈現。
7. **內容補齊**:照 30 單元地圖,逐課補四式操練 + 母語錨點 (`nativeZh`)。

> **目前雛形狀態**:30 單元學習地圖 + 推進(localStorage)、替換/轉換兩層選句框、擴展抽5輪替、瀏覽器 VAD(TTS 不被切)、Echo Loop(含覆述停頓)、後台 `/admin`(句框/單字)、HTTPS 區網(手機可連)。
> **AI 評分**:STT(`/api/stt`)+ 評分(`/api/evaluate`)程式已接好,**填 `.env.local` 的 `OPENAI_API_KEY` 並重啟即啟用**;未設則自動免費模式。
> **自由發揮模式**:已寫入規格,**程式尚未實作**(待辦)。
> 內容目前 **7 課**:Unit 1–6 + Unit 15(Unit 7–10 與第二/三週期待補)。

- [ ] 答錯/反應慢的會進弱點頁,隔天再出現。
- [ ] 弱點頁能點選直接操練。
- [ ] 進度頁看得到練習天數、正確率、平均反應速度。
